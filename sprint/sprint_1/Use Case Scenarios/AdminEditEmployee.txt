Context - User is logged in as an admin account

Primary Flow
1  - Admin navigates to View/Create/Edit/Delete Employee Page on WebApp
2  - System navigates admin to view of all users with in application 
3  - Admin searches and selects employee to edit
4  - System navigates admin to edit employee page
5  - Admin inputs edited credentials (username, password, role)
6  - Admin select update employee credentials button
7  - System validates updated credentials
8  - System updates database with new credentials
9  - System notifies Admin of successful editing of existing credentials
10 - System navigates admin back to View/Create/Edit/Delete Employee Page 

Alternate Flow(s)
ADMIN INPUTS INVALID CREDENTIALS 
a1 - Start at Primary Flow step 4
a2 - Admin does not enter all required credentials or invalid credentials
a3 - System notifies admin of expected requirements with error label
a4 - System reprompts usser to re-enter edited credentials
a5 - Back to Primary Flow step 5
